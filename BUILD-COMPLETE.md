## What works end-to-end

When you complete the README setup, you'll be able to:

1. **Sign in** via magic link to your email
2. **See Silas's opening message** — generated by him reading 20 of your real notes
3. **Talk to him** — every message hits the chat API, which:
   - Retrieves the 8 most relevant notes from your vault (RAG)
   - Retrieves the 5 most relevant past message snippets (total recall)
   - Builds the full system prompt with personality + context
   - Streams Claude's response back live (token-by-token)
4. **See it capture notes** — after every exchange, Haiku silently extracts anything worth saving and writes it to your vault
5. **Have it remember everything** — every message you've ever sent is searchable via past-message RAG

## What I'm NOT sure works (verify-before-claim)

Per the `verify-before-claim` skill, full honesty about untested parts:

- ❓ **Migration script edge cases** — tested logic mentally, didn't run against your real vault. Likely needs adjustment for: very large notes (>30KB triggers OpenAI truncation), notes with broken frontmatter, files with special chars in paths.
- ❓ **Magic link flow** — wrote it per Supabase docs but didn't actually test sign-in. First attempt may need redirect URL tweaks in Supabase dashboard.
- ❓ **Streaming error states** — happy path tested in code. If Anthropic rate-limits or times out mid-stream, the UI just hangs (no retry yet).
- ❓ **RLS policies** — designed for single user. If you ever add a second user, audit them.
- ❓ **Vercel cold starts** — first request after idle is slow. Free tier limitation.
